Browser-side parse: FileReader reads CSV text, detects format (32-col rows vs 1024-col rows), renders first frame on canvas preview, counts frames and estimates session duration — all before upload.

Key design decisions

SensorFrame.data stores the 32×32 matrix as a flat JSON array of 1024 integers rather than a normalised SensorPixel table. This avoids 1024 rows per frame (900,000 rows for 5 patients × 3 sessions × 60 frames) and keeps reads fast — a single column fetch returns the whole frame.

PressureMetrics.hot_zones stores the top-5 high-pressure pixel coordinates as a JSON array [{x, y, value}]. These are used to draw the animated ring markers on the heatmap canvas without re-scanning the matrix on the frontend.

Comment.reply_to is a self-referential FK. The API filters is_reply=False for top-level comments and then fetches replies as a nested list, giving the threaded display without a separate Reply table.
